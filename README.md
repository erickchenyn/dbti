# DBTI — Dota2 Behavioral Type Indicator

> **创建者**：ChenYn & ChenYn's momo · **创建时间**：2026-04-10 · **最后更新**：2026-04-10

## 概述

参考 [SBTI 人格测试](https://sbti.unun.dev/) 的交互形式，设计一款 Dota2 玩家行为类型测试。通过围绕 Dota2 游戏场景设计的选择题，生成玩家的 **功能画像**：适合的位置、擅长的事情、队伍中的角色定位。

**定位**：娱乐向，带有 Dota2 社区特有的自嘲幽默感。

## 评测框架

### 核心结构

评测由三层结构组成：**模型 → 维度 → 等级**。

| 层级 | 说明 | 当前配置 |
|------|------|----------|
| 模型（Model） | 描述玩家行为的一个大类切面 | 5 个 |
| 维度（Dimension） | 模型下的具体衡量指标，每个维度对应若干道题 | 每模型 3 个，共 15 个 |
| 等级（Level） | 每个维度的得分档位 | 3 档：L / M / H |

### 题目与计分

| 参数 | 值 | 说明 |
|------|-----|------|
| 每维度题数 | 2 | 每个维度由 2 道选择题衡量 |
| 每题分值 | 1 / 2 / 3 | 每道题 3 个选项，对应 1-3 分 |
| 维度得分范围 | 2 - 6 | = 每维度题数 × 每题分值范围 |
| 总题数 | 30 + 1 隐藏 | 15 维度 × 2 题 + 隐藏题 |

### 等级划分

维度得分会被映射为 L / M / H 三个等级，划分方式为将得分范围均分为 `LEVEL_COUNT` 段：

| 等级 | 代号 | 得分区间 |
|------|------|----------|
| 低 | L | 2 - 3 |
| 中 | M | 4 |
| 高 | H | 5 - 6 |

### 类型匹配算法

1. 每种人格类型预设一个 **15 维向量**，每个维度值为 1（L）/ 2（M）/ 3（H）
2. 用户答完所有题目后，将各维度得分映射为等级，得到用户的 15 维向量
3. 计算用户向量与所有类型向量的 **欧氏距离平方和**
4. 距离最近的类型为 **主类型**，次近为 **副类型**，同时给出匹配度百分比
5. 匹配度 = `(1 - 距离 / 最大可能距离) × 100%`

### 隐藏题机制

当 Sy2（胜负心）两题均选最高分（维度得分达到上限）时，触发 1 道隐藏题。若选择特定选项，解锁隐藏类型 **TILT**，覆盖正常匹配结果。

### 扩展原则

#### 新增题目

- 每道题必须归属一个明确的维度（`dim`），且只衡量该维度，不应同时测量多个维度
- 每道题提供 3 个选项，分值分别对应 1（低分极倾向）、2（中间）、3（高分极倾向）
- 题目应基于具体的 Dota2 游戏场景，避免抽象或脱离游戏的表述
- 同一维度下的多道题应从不同场景切入（如对线期、团战期、逆风期），避免场景雷同
- 新增题目后需同步更新 `QUESTIONS_PER_DIM`（若每维度题数变化），得分区间和等级划分会自动重新计算
- 选项文本应保持口语化和幽默感，符合 Dota2 社区语境

#### 新增模型或维度

- 新模型应代表一个独立的行为切面，与已有模型（战略、资源、团战、心态、风格）不重叠
- 新维度应在所属模型内衡量一个独立的行为特征，低分极和高分极应形成清晰的两端对立
- 新增维度后，所有人格类型的预设向量都必须补充该维度的值（1/2/3），否则匹配算法无法工作
- 建议每个模型下维度数量保持一致，以保证各模型在匹配算法中的权重均衡

1. **类型名 + 匹配度** — 大标题展示
2. **一句话标签** — 类型的核心性格描述
3. **详细描述** — 2-3 段文字，用 Dota2 场景讲故事
4. **位置推荐** — ✅/⚠️/❌ 标注 1-5 号位 + 队长/BP 的适配度
5. **擅长 / 不擅长** — 各 3-4 条，涵盖具体游戏行为
6. **15 维度雷达图** — 可视化各维度得分
7. **队友评价** — 一段虚构的队友吐槽/夸赞语录
8. **副类型** — "你的隐藏面：XX"，展示第二匹配类型

## 交互流程

```
[开屏] "你是什么型的Dota2玩家？"
  ↓ 点击"开始测试"
[答题] 30题逐题滑动，每题3个选项
  ↓ 触发隐藏题条件 → 插入隐藏题
[计算] 加载动画（模拟匹配中...）
  ↓
[结果] 展示完整画像
  ↓
[分享] 生成分享卡片（类型名 + 一句话标签 + 位置推荐）
```

## 视觉风格

- **主色调**：Dota2 暗红 + 深灰，搭配金色高亮
- **字体**：标题用粗体无衬线，正文保持可读性
- **图标**：使用 Dota2 风格的简笔图标（盾牌、剑、眼、烟等）
- **动效**：答题切换使用横向滑动，结果页使用维度条逐条动画展开

## 技术说明

- 纯前端实现，单个 HTML 文件，无需后端
- 所有题目和类型数据内嵌在 JS 中
- 使用 CSS 动画实现过渡效果
- 雷达图使用 SVG 绘制
- 分享功能使用 html2canvas 生成图片
